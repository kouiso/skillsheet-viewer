# Gemini Code Assist スタイルガイド

## 基本方針

### 言語
- 日本語でレビューコメント・提案を記述する
- コード内のコメントも日本語を推奨

### コードスタイル
- 既存のコードスタイル・設計パターンに従う
- プロジェクトのリンター・フォーマッター設定を尊重する

### レビュー方針
- バグ・セキュリティ問題を最優先で指摘
- パフォーマンス改善の提案
- 可読性・保守性の向上提案

## ペルソナ
- 対話時のペルソナや口調は `prompt/prompt.md` の指示を参照する
- 個人の健康・医療・障害などのセンシティブ情報は、このファイルへ転載しない
- 参照先の情報を扱う場合も、必要最小限の文脈に限定し、レビュー対象の品質向上に関係する内容だけを使う

## トリビア
- 雑談や小ネタは、レビューの主題を邪魔しない範囲で短く添える
- 個人を特定できる詳細情報やセンシティブ情報をトリビアとして扱わない
- 技術的な比喩を使う場合は、誤解を招かないように結論を先に書く

## ポエム
- 励ましや締めの一言は、作業者の次の行動が明確になるように書く
- 過度に長い情緒的表現は避け、レビューコメントの実用性を優先する
- 機密情報・個人情報・センシティブ情報を詩的表現に含めない
